# 🕵️ 마니또 추리 대회

마니또 배정을 **누가 누구인지 맞혀보는 추리 대회** 웹앱입니다.
참가자들이 "전체 배정표"를 예측해 제출하고, 발표일에 운영자가 채점해 순위를 봅니다. **조별로 완전히 분리**돼서 한 앱으로 여러 조가 동시에 써요.

**라이브:** https://maeumbaedal-quiz.vercel.app

> 짝꿍 앱: [마음배달(봉투)](https://github.com/baezzanglion/maeumbaedal) — 마니또를 뽑아 개인 봉투 링크를 나눠주는 앱

---

## 어떻게 작동하나요

- **방(room) 단위**로 운영돼요. URL의 `?room=` 값으로 조를 구분합니다. (예: `1jo`, `2jo`, …)
- 각 조 **운영자**가 어드민 화면에서 **참가자 명단**을 등록합니다.
- **참여자**는 자기 조 링크로 들어가 "○○의 마니또는 △△일 것이다"를 전부 예측해 제출합니다.
- **정답(실제 배정)은 서버에 저장하지 않습니다.** 발표일에 운영자가 정답을 넣으면 **브라우저에서만** 채점해 순위를 냅니다 → 익명성 보장.

---

## 링크 구조

| 종류 | 주소 |
|---|---|
| 참여자 | `https://maeumbaedal-quiz.vercel.app/?room=1jo` |
| 어드민 | `https://maeumbaedal-quiz.vercel.app/?room=1jo&admin=1` |

`room` 값(소문자·숫자·`-`·`_` 1~32자, 예: `1jo`, `2jo`, `abc`)만 바꾸면 방이 무한히 생겨요. **방끼리는 서로 안 보입니다.**

---

## 사용법

**운영자**
1. 어드민 링크(`?room=…&admin=1`) 접속
2. 참가자 명단(한 줄에 한 명) + 조 이름 등록
3. 참여자 링크를 조원들에게 공유
4. 발표일에 어드민 화면에서 정답 입력 → 순위 확인

**참여자**
1. 참여자 링크(`?room=…`) 접속
2. 각 사람의 마니또를 예측해서 제출 (다시 제출하면 갱신됨)

---

## 다른 조가 복제해서 쓰는 법

이 앱은 **Vercel serverless + Blob 저장소**가 필요해요 (명단·예측을 저장하니까).

```bash
git clone https://github.com/baezzanglion/maeumbaedal-quiz.git
cd maeumbaedal-quiz
npm install
npx vercel link            # 자기 Vercel 프로젝트에 연결
# → Vercel 대시보드에서 Blob store 생성 후 프로젝트에 연결하면
#   BLOB_READ_WRITE_TOKEN 환경변수가 자동으로 붙어요
npx vercel deploy --prod
```

- **핵심:** 반드시 **자기 Vercel + 자기 Blob store**에 배포하세요. 그래야 데이터가 완전히 분리됩니다.
- 방 이름은 URL에서 자유롭게 정하면 돼요. 명단은 미리 넣을 필요 없이 어드민 화면에서 등록합니다.

## 기술 구조

- `index.html` — 참여자/어드민 화면 (단일 파일 SPA)
- `api/roster.js` — 조 명단 조회/저장 (`rooms/<room>.json`)
- `api/submit.js` — 참여자 예측 저장 (`p/<room>/<이름해시>.json` — 이름을 해시로 익명 키 처리)
- `api/data.js` — 방의 제출 목록 조회 (채점용)
- 저장소: Vercel Blob (`@vercel/blob`)

## 개인정보 · 익명성

- **실제 마니또 정답은 서버에 저장하지 않습니다.** 참가자는 자기 예측만 올리고, 채점은 발표일에 운영자 브라우저에서만 이뤄져요.
- 제출은 이름 해시를 키로 저장돼 방 안에서 재제출 시 갱신됩니다.
- 방(room)마다 데이터가 분리돼 다른 조의 예측은 서로 보이지 않아요.
